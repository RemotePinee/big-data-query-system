## 🎯 系统概述

**大数据查询系统**是一个现代化的数据查询平台，提供高效的数据查询、分析和可视化功能。系统采用前后端分离架构，支持PC端和移动端访问。

### ✨ 主要特性
- 🔐 **安全认证**：JWT身份认证，多层权限控制
- 📊 **数据查询**：支持多种数据源和查询类型
- 💳 **在线支付**：集成微信支付、支付宝、易支付
- 📱 **响应式设计**：完美支持PC端和移动端
- 👨‍💼 **管理后台**：功能完善的管理员控制面板
- 📈 **数据统计**：实时统计和趋势分析

### 🌐 访问地址
- **前端地址**：http://localhost:5176
- **管理后台**：http://localhost:5176/admin
- **API接口**：http://localhost:3000/api
- **系统健康检查**：http://localhost:3000/api/health

## 🚀 快速开始

### 环境要求
- Node.js >= 16.0.0
- MySQL >= 5.7
- Redis >= 6.0 (可选，用于缓存)

### 启动系统

#### 方法一：使用启动脚本（推荐）
```bash
# 开发环境
双击 start-development.bat

# 生产环境
双击 start-production.bat
```

#### 方法二：手动启动
```bash
# 1. 启动后端
cd backend
npm install
npm run dev

# 2. 启动前端
cd frontend
npm install
npm run dev
```

### 初次使用配置
1. 配置数据库连接（修改 `backend/.env` 文件）
2. 运行数据库初始化脚本
3. 创建管理员账户
4. 配置支付参数（可选）

## 👥 用户端功能

### 1. 用户注册与登录

#### 注册新账户
1. 访问首页，点击"注册"按钮
2. 填写注册信息：
   - 用户名（必填，3-20个字符）
   - 密码（必填，6-20个字符）
   - 邮箱（可选，用于找回密码）
   - 手机号（可选）
3. 点击"注册"完成账户创建

#### 用户登录
1. 点击"登录"按钮
2. 输入用户名和密码
3. 系统自动保存登录状态

### 2. 数据查询功能

#### 查询流程
1. **选择查询类型**
   - 浏览可用的查询项目
   - 查看价格和说明
   - 选择需要的查询类型

2. **输入查询参数**
   - 根据查询类型填写必要参数
   - 系统会自动验证参数格式
   - 预览查询费用

3. **提交查询请求**
   - 确认查询信息和费用
   - 选择支付方式
   - 完成支付后开始查询

4. **查看查询结果**
   - 查询完成后系统自动通知
   - 在"个人中心"查看结果
   - 支持结果下载和导出

#### 查询类型说明
- **基础查询**：免费或低价格的基础数据查询
- **高级查询**：付费的详细数据分析
- **批量查询**：支持批量数据处理
- **定制查询**：根据需求定制的专业查询服务

### 3. 个人中心管理

#### 账户信息
- 查看和修改个人资料
- 头像上传和更换
- 密码修改和安全设置
- 绑定手机号和邮箱

#### 订单管理
- 查看所有查询订单
- 订单状态跟踪：
  - `待支付`：需要完成支付
  - `查询中`：正在处理查询
  - `已完成`：查询完成，可查看结果
  - `已取消`：订单已取消

#### 查询历史
- 浏览历史查询记录
- 重新下载查询结果
- 查看查询详情和参数

#### 账户余额
- 查看账户余额
- 充值记录查询
- 消费明细查看

## 💳 支付系统

### 支付流程
1. **选择查询服务**：选择需要的付费查询项目
2. **确认订单**：确认查询参数和费用
3. **选择支付方式**：微信支付、支付宝、易支付
4. **完成支付**：跳转到支付页面完成付款
5. **支付确认**：系统确认支付后开始查询
6. **查看结果**：查询完成后查看结果

### 支付方式

#### 微信支付
- 支持微信扫码支付
- 支持微信内H5支付
- 实时到账，自动确认

#### 支付宝
- 支持支付宝扫码支付
- 支持支付宝网页支付
- 快速到账处理

#### 易支付
- 支持多种银行卡支付
- 支持第三方支付平台
- 灵活的支付配置

### 支付安全
- **加密传输**：所有支付数据采用HTTPS加密
- **签名验证**：支付回调采用数字签名验证
- **实时监控**：支付过程实时监控和日志记录
- **风险控制**：异常支付自动拦截和提醒

## ❓ 常见问题

### 用户常见问题

**Q: 忘记密码怎么办？**
A: 联系管理员重置密码，或使用邮箱找回功能（如已绑定邮箱）。

**Q: 查询失败怎么办？**
A: 
1. 检查查询参数是否正确
2. 确认账户余额是否充足
3. 联系技术支持获取帮助

**Q: 支付成功但订单状态未更新？**
A: 
1. 等待1-2分钟系统自动处理
2. 刷新页面查看最新状态
3. 如仍有问题请联系客服

**Q: 如何下载查询结果？**
A: 在个人中心的查询历史中，点击对应订单的"下载结果"按钮。

### 管理员常见问题

**Q: 如何添加新的查询项目？**
A: 进入管理后台 → 查询项目管理 → 点击"添加项目" → 填写项目信息 → 保存。

**Q: 如何配置支付接口？**
A: 进入管理后台 → 支付配置 → 选择支付方式 → 填写接口参数 → 测试连接 → 保存配置。

**Q: 用户订单异常如何处理？**
A: 进入订单管理 → 找到异常订单 → 查看详情 → 根据情况进行退款或重新处理。

### 技术问题

**Q: 系统启动失败？**
A: 
1. 检查Node.js版本是否符合要求
2. 确认数据库连接配置正确
3. 查看错误日志定位问题
4. 检查端口是否被占用

**Q: 数据库连接失败？**
A: 
1. 检查数据库服务是否启动
2. 确认数据库连接参数正确
3. 检查数据库用户权限
4. 确认防火墙设置
